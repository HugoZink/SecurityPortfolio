1. Spoofing the ASP.NET Core App Process  [State: Mitigation Implemented]  [Priority: High] 

Category:Spoofing
Description:Search Accommodations (ASP) may be spoofed by an attacker and this may lead to information disclosure by User. Consider using a standard authentication mechanism to identify the destination process.
Justification:HTTPS has been implemented for communication between the User and the ASP.NET Core webserver. Also see OWASP Top 10 2017 #3: Sensitive Data Exposure. HTTPS effectively eliminates man-in-the-middle attacks. The relevant security goals are Confidentiality and Integrity.
Short Description:Spoofing is when a process or entity is something other than its claimed identity. Examples include substituting a process, a file, website or a network address.

Interaction: HTTP

2. Potential Lack of Input Validation for Search Accommodations (Node)  [State: Mitigation Implemented]  [Priority: High] 

Category:Tampering
Description:Data flowing across HTTP may be tampered with by an attacker. This may lead to a denial of service attack against Search Accommodations (Node) or an elevation of privilege attack against Search Accommodations (Node) or an information disclosure by Search Accommodations (Node). Failure to verify that input is as expected is a root cause of a very large number of exploitable issues. Consider all paths and the way they handle data. Verify that all input is verified for correctness using an approved list input validation approach.
Justification:A sanitization library has been implemented for all possible query parameters within the accommodation search API function. Any kind of SQL Injection should now be prevented. Also see OWASP Top 10 2017 #1: Injection. The relevant security goal is Integrity.
Short Description:Tampering is the act of altering the bits. Tampering with a process involves changing bits in the running process. Similarly, Tampering with a data flow involves changing bits on the wire or between two running processes.

Interaction: HTTP

3. Data Flow Sniffing  [State: Mitigation Implemented]  [Priority: High] 

Category:Information Disclosure
Description:Data flowing across HTTP may be sniffed by an attacker. Depending on what type of data an attacker can read, it may be used to attack other parts of the system or simply be a disclosure of information leading to compliance violations. Consider encrypting the data flow.
Justification:HTTPS has been implemented in the Node API. Data flow sniffing and man in the middle attacks are no longer possible. The relevant security goals are Confidentiality and Integrity.
Short Description:Information disclosure happens when the information can be read by an unauthorized party.

Interaction: HTTP

4. Potential Lack of Input Validation for Login (Node)  [State: Mitigation Implemented]  [Priority: High] 

Category:Elevation Of Privilege
Description:Elevation of privilege can be achieved on Login (Node), by brute-forcing the login process. This may lead to an attacker gaining unwanted privileges.
Justification:By rate limiting the amount of login requests from one machine, brute-forcing essentially becomes impossible. An attacker only gets a few tries at a time, after which the attacker is locked out for a period of time. To maintain Availability for real users, the account itself is not locked. Also see OWASP Top 10 2017 #2: Broken Authentication. The relevant security goals are Integrity and Authentication.
Short Description:Tampering is the act of altering the bits. Tampering with a process involves changing bits in the running process. Similarly, Tampering with a data flow involves changing bits on the wire or between two running processes.

Interaction: HTTP

5. Potential Lack of Input Validation for Add Accommodation (Node)  [State: Mitigation Implemented]  [Priority: High] 

Category:Tampering
Description:Data flowing across HTTP may be tampered with by an attacker. This may lead to a denial of service attack against Create Accommodation (Node) or an elevation of privilege attack against Create Accommodation (Node) or an information disclosure by Create Accommodation (Node). Failure to verify that input is as expected is a root cause of a very large number of exploitable issues. Consider all paths and the way they handle data. Verify that all input is verified for correctness using an approved list input validation approach.
Justification:It was possible for an attacker to add HTML tags to any of the Accommodation&#39;s properties (name, description, etc). In turn, this could cause an XSS attack. By stripping the HTML out of all user-entered data, this attack has been effec tively mitigated. Also see OWASP Top 10 2017 #7: XSS. The relevant security goal is Integrity.
Short Description:Tampering is the act of altering the bits. Tampering with a process involves changing bits in the running process. Similarly, Tampering with a data flow involves changing bits on the wire or between two running processes.

6. External Entity Human User Potentially Denies Receiving Data  [State: Mitigation Implemented]  [Priority: High] 

Category:Repudiation
Description:Human User claims that it did not receive data from a process on the other side of the trust boundary. Consider using logging or auditing to record the source, time, and summary of the received data.
Justification:User actions are logged whenever they create or update an accommodation. This log can prove that the user did indeed create an accommodation. Also see OWASP Top 10 2017 #10: Insufficient Logging &amp; Monitoring. The relevant security goal is Non-Repudiation.
Short Description:Repudiation threats involve an adversary denying that something happened.
